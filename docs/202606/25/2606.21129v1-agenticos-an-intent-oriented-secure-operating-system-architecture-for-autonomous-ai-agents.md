## 摘要
随着大语言模型（LLM）驱动的自主代理具备规划、工具使用、网络访问和代码执行能力，基于“资源暴露加权限检查”的传统操作系统安全模型面临结构性挑战。一旦代理运行时通过提示注入或恶意工具输出被攻破，攻击者可将POSIX风格的原语组合成远超用户任务授权的行为。为此，我们提出AgenticOS，一种意图导向的安全操作系统架构，它将可委托、可审计的软件能力整合为操作系统原生能力，而非替换所有应用。核心洞见在于将操作系统从“资源管理器”重塑为“意图过滤器”：代理不直接请求底层资源，而是提交结构化的意图声明，系统据此综合出一个最小权限环境，并施加强制中介、审计和信息流约束。在实现层面，我们引入了四层架构——幽灵内核、逻辑快门、代理胶囊和语义边界网关——以及意图ABI、仅依赖清单的运行时、基于编织器的能力生成，和针对AgenticOS原生技能的准入模型。

## Abstract
Traditional OS security models based on "resource exposure plus permission checks" face structural challenges as LLM-driven autonomous agents acquire capabilities for planning, tool use, network access, and code execution. Once an agent runtime is compromised through prompt injection or malicious tool outputs, an attacker can compose POSIX-style resource primitives into behaviors far beyond the user's task authorization. To address this, we propose AgenticOS, an intent-oriented secure OS architecture that consolidates delegable, auditable software capabilities into OS-native ones rather than replacing all applications. The core insight is to reframe the OS from a "resource manager" into an "intent filter": instead of requesting low-level resources directly, agents submit structured intent declarations, from which the system synthesizes a least-privilege environment with mandatory mediation, auditing, and information-flow constraints. At the implementation level, we introduce a four-layer architecture -- Ghost Kernel, Logic Shutter, Agent Capsule, and Semantic Boundary Gateway -- together with the Intent ABI, Manifest-Only Runtime, Weaver-based capability generation, and an admission model for AgenticOS-native Skills.

## 1. 核心问题与整体含义
- **研究动机**：LLM驱动的自主代理（Agent）具备规划、工具调用、网络交互和代码执行能力，其运行模式从“用户操作应用”转变为“用户给定高层任务，代理自主分解执行”。传统操作系统基于“资源暴露 + 权限检查”（POSIX系统调用如`open, write, socket, execve`）的安全模型面临根本挑战。
- **核心安全问题**：代理运行时若被提示注入、供应链投毒或恶意工具输出攻破，攻击者可将大量中性但组合性极强的资源原语（文件、网络、进程、动态执行）拼接成越权行为（横向移动、数据窃取、持久化等），远超用户初始任务授权。这被称为 **“资源权限模型与任务意图之间的错配”**。
- **整体含义**：操作系统安全模型需从“能否访问某个资源”转向“所产生的对外效应是否符合声明的任务意图”。AgenticOS将OS重构为 **“意图过滤器”**，而非资源管理器，旨在为高度自主的AI系统提供以意图约束和最小能力为核心的安全计算基座。

## 2. 方法论：核心思想与关键技术细节
- **核心范式转换**：操作系统不再是资源管理器，而是意图过滤器。代理运行时不再直接依赖通用系统调用语义，而是通过 **意图声明（Manifest）** 提交任务边界，系统据此一次性综合出最小能力环境，并实施强制中介、审计和信息流控制。
- **四层架构**（严格单向依赖，纵深防御）：
  - **Ghost Kernel（幽灵内核）**：最小化可信隔离基座，运行在最高特权级（VMM Root/TEE）。仅提供加密内存分配、确定性时间片调度、度量与证明根。不暴露任何面向代理胶囊的运行时接口，从而结构性地移除以系统调用为入口的提权路径。
  - **Logic Shutter（逻辑快门）**：语义翻译与策略执行层，负责意图解析与验证、不可伪造的能力令牌（capability token）管理、结构化审计日志生成以及信息流标签实施。所有从胶囊发出的语义请求均在此匹配Manifest中声明的能力、数据流约束和输出目标。
  - **Agent Capsule（代理胶囊）**：受限的代理运行时。遵循 **“Manifest-Only Runtime”原则**：启动时提交Manifest，系统通过Weaver动态生成专用ABI表；Manifest未声明的能力在胶囊地址空间中完全没有对应接口。胶囊无直接暴露`fork, exec, mmap, dlopen, socket`等通用执行或I/O原语。
  - **Semantic Boundary Gateway（语义边界网关）**：外部世界的唯一受控边界，负责协议代理（TLS、HTTP等不暴露给胶囊）、凭证托管、结构化消息通道（严格模式校验防协议隧道），以及输出审计与归一化（内容过滤、大小填充、节奏控制）。
- **Intent ABI（意图应用二进制接口）**：
  - 取代传统syscall ABI，提供六大类结构化语义操作：身份与会话、文档与数据、网络能力、工具链、模型推理、环境观测。
  - 设计原则：**非通用化**（组合闭包不能退化为通用系统调用表）、**无字节流**（禁止socket/pipe/raw fd）、**无任意执行**（工具链能力分解为可审计语义接口）、**物化能力**（网络权限绑定五元组）、**可审计输出**、**受约束信息流**（接口输入输出携带标签）。
  - 永久排除的接口：`open, read, write, socket, connect, bind, mmap, execve, fork, ptrace, dlopen, syscall`等，作为静态分析、链接器及形式化验证的硬终止规则。
- **Weaver与动态能力生成**：
  - Weaver在构建时将Manifest、代理代码、标准语义库链接成胶囊运行时环境，并必须验证：代码中不存在未声明能力的调用点；标准库签名合法且版本可溯；无禁止接口可达；信息流路径符合策略；构建可复现。
- **AgentOS原生技能生态与准入模型**：
  - 技能（Skill）指可通过Intent ABI调用的系统原生能力单元（如“整理表格生成报告”）。新技能的生成仅允许在开发态，并需通过声明、静态分析、测试/证明、签名注册等准入流程；运行时只能通过Manifest声明和能力令牌调用已注册技能，防止代理自升级能力。

## 6. 主要结论与发现
- **架构可行性**：通过将OS重建为意图导向的四层安全系统，可以从根本上改变代理的安全边界：代理不再直接获取原始文件、网络、进程、协议能力，而仅通过Manifest、Intent ABI、能力令牌、逻辑快门和语义边界网关产生受控的外部效应。
- **攻击面重构**：攻击者不再能够组合通用系统调用实现越权，只能在一个更窄的、高度审计和策略约束的抽象层内操作，这大幅增加了攻击难度并强化了可追溯性。
- **生态迁移路径**：长期目标并非替换所有应用，而是将可委托、可语义抽象、可审计的软件能力逐步固化为操作系统原生技能，实现从“应用功能”到“系统侧能力对象”的迁移，同时保留必要的人机界面和人类确认机制。
- **安全边界与责任**：AgenticOS不承诺完全消除风险（如隐蔽信道、意图不可判定性），但可以在架构层面将高风险能力与人类责任锚定（如RequestHumanApproval作为一等公民），明确自主代理与人类责任的技术分界。
