## 摘要
我们提出CHERI-D，这是一种CHERI的架构扩展，支持高效的时间内存安全。高效的内存安全对于编程语言、操作系统和硬件设计日益重要，而CHERI是领先的硬/软件系统，提供原生的空间安全并为时间内存安全奠定基础。由于CHERI缺乏对时间内存安全的内在架构支持，最先进的CHERI时间安全解决方案Cornucopia Reloaded是一种基于软件的方案，提供释放后重分配（UAR）保护而不是更强的释放后使用（UAF）缓解，并因延迟重分配和撤销而导致性能开销。CHERI-D将对象标识（ID）元数据与能力指针关联，以提供分配的时间完整性。CHERI的空间安全允许CHERI-D将对象ID安全地内联存储在分配数据中，可能利用未使用的碎片。在仿真和硬件中评估，CHERI-D显著降低了Cornucopia Reloaded的撤销开销，同时使其能够支持严格的释放后使用缓解。

## Abstract
We propose CHERI-D, an architectural extension to CHERI that supports efficient temporal memory safety. Efficient memory safety is an increasing priority for programming languages, operating systems, and hardware designs, and CHERI is a leading hardware/software system that provides native spatial safety and a foundation for temporal memory safety. Due to CHERI lacking intrinsic architectural support for temporal memory safety, the state-of-the-art CHERI temporal safety solution, Cornucopia Reloaded, is a software-based solution that provides use-after-reallocation (UAR) protections instead of the stronger use-after-free (UAF) mitigation, and suffers performance overhead due to delayed reallocation and revocation. CHERI-D associates object identification (ID) metadata with capability pointers to provide temporal integrity of allocations. CHERI spatial safety allows CHERI-D to store object IDs safely inline with allocation data, potentially within unused fragmentation. Evaluated in simulation and in hardware, CHERI-D significantly reduces the revocation overhead of Cornucopia Reloaded while allowing it to support strict use-after-free mitigation.

*   **核心问题**： CHERI 架构提供了强大的原生空间内存安全（通过有界指针），但缺乏对**时间内存安全**（即防止释放后使用 Use-After-Free, UAF 漏洞）的硬件级支持。
*   **现有方案与局限**： 当前最先进的 CHERI 时间安全解决方案是 Cornucopia Reloaded，但它存在几个关键局限：
    *   **安全保证较弱**： 它主要缓解的是**释放后重分配（UAR）** 攻击，而非更强的**释放后使用（UAF）** 攻击。因为它在内存被释放后会将其隔离，在完成一次全局地址空间扫描以撤销所有悬空指针之前，对该内存的访问仍被允许，这一时间窗口仍可被利用。
    *   **性能开销大**： 延迟重分配和频繁的全局撤销扫描带来了显著的性能损失，包括内存占用增加、缓存污染、DRAM 带宽损耗以及破坏了现代内存分配器基于局部性原理的优化（如线程缓存）。
*   **整体意义**： 论文旨在为 CHERI 系统设计一种硬件扩展，以高效地提供严格的 UAF 保护，解决软件方案的性能瓶颈和安全保证不足的问题。

### 2. 论文提出的方法论

CHERI-D 的核心思想是为内存分配引入一个**架构级保护的对象标识符 (Object ID)**，并将其安全地内联存储在分配的数据块中，从而追踪内存的生命周期。

#### 核心思想与关键技术细节

*   **能力扩展**： CHERI-D 在 CHERI 的能力 (Capability) 元数据中新增了以下字段：
    *   `capability ID` (8位): 代表该能力有权访问的内存生命周期（版本号）。
    *   `IDMODE` (1位) 和 `IDLOC` (6位): 共同编码了内存中对象 ID 的存储位置。
*   **内联 ID 存储 (Inline ID Storage)**： 利用 CHERI 自身的界限检查能力，将信任的 ID 元数据直接存储在分配数据块内部，但位于能力界限之外，从而在保护 ID 不被用户程序篡改的同时，避免了传统标签方案所需的外部影子内存及其开销。
    *   **In-line ID 模式** (适用于小于64字节/单缓存行的分配): ID 直接放置在缓存行内分配的末尾。ID查找与数据访问同时发生，几乎零额外延迟。
    *   **In-page ID 模式** (适用于小于4KiB页面但跨缓存行的分配): 在物理页的顶部维护一个紧凑的 ID 表（64个ID/64字节）。利用一个小的 **ID 缓冲区 (ID Buffer)** 来缓存这些ID，实现早期的ID验证。
*   **立即重用与版本管理**： 内存释放后，可被立即重新分配并赋予一个新的、递增的 ID（最多255次重用）。通过比较能力中的 ID 和内存中的 ID 来验证访问合法性，不匹配即触发异常，实现了严格的 UAF 保护。
*   **安全授权模型**： 特权代码（如分配器）拥有修改 `ID`、`IDLOC` 和 `IDMODE` 的能力，而用户态能力一旦被赋予非零 ID，就无法再修改这些字段，确保了元数据的完整性。
*   **撤销机制简化**： 保留一个特殊的 ID 值（255）来标记被隔离的内存。撤销扫描器通过探测能力中的 ID 和内存中的 ID 来判断能力是否失效，不再依赖 Cornucopia 的影子位图，减少了分配器在释放路径上的开销。
