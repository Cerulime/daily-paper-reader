## 摘要
深度神经网络和大语言模型（LLM）的广泛部署基于一个根本性的隔离假设，而本论文将对此提出挑战。这项工作通过一系列交互层面的分类，从物理内存共置到远程服务接口，系统性地解构了围绕人工智能和现代云基础设施的安全假设。尽管已有大量研究分别针对各个攻击面，安全社区仍缺乏一个统一的框架，用以推理物理、架构和算法层面的漏洞如何在整个现代人工智能技术栈中显现。本论文通过展示利用每一抽象层假设的实际攻击，弥补了这一空白。

## Abstract
The widespread deployment of Deep Neural Networks and Large Language Models (LLMs) relies on a foundational assumption of isolation that this dissertation challenges. This work systematically deconstructs security assumptions around AI and modern cloud infrastructure through a taxonomy of interaction levels that ranges from physical memory co-location to remote service interfaces. While significant research has addressed individual attack surfaces in isolation, the security community lacks a unified framework for reasoning about how physical, architectural, and algorithmic vulnerabilities manifest across the modern AI stack. This dissertation addresses that gap by demonstrating practical attacks that exploit assumptions at each layer of abstraction.

### 2. 论文提出的方法论

论文将攻击面分为三个交互层次，并为每个层次设计了独特的攻击方法。

**层次一：共享内存访问（攻击者可读取共享内存区域）**
- **“Spill The Beans”攻击（第3章）**：
  - **核心思想**：利用 Flush+Reload 缓存侧信道技术，监控 LLM 的嵌入层（Embedding Layer）访问模式，以逐 token 重建用户的私有提示词。
  - **关键技术细节**：攻击者与受害者进程共置，利用内存页合并（KSM）共享 GGUF 模型文件。通过循环监控目标 token 对应的嵌入向量偏移量，测量访问时间（RDTSC）以判断是否命中缓存，从而推断出具体 token。为解决大词表带来的缓存逐出问题，采用轮询（Round-robin）策略并优化监控的 token 数量（约 150-250 个）以平衡覆盖与漏检。
- **“RubberMallet”攻击（第4章）**：
  - **核心思想**：利用 Rowhammer 在现代 DRAM 中引发的高频“相邻比特翻转”现象，攻击大语言模型的词表文件（Tokenizer Dictionary）或加密算法（ECDSA）。
  - **关键技术细节**：
    - **词表污染**：通过精确的比特翻转，交换 GGUF 词表中 token ID 对应的字符串，从而在不修改模型权重的前提下，重写系统提示词或安全指令，绕过安全护栏。
    - **密码攻击**：利用相邻比特翻转在 ECDSA 签名随机数中产生的可预测算术关系（如 Δ = ±3·2ⁱ），结合改进的格基约减算法（Bounded-Distance Decoding with Predicate）恢复私钥。

**层次二：共享硬件资源访问（无显式共享内存，仅有 CPU 等硬件共置）**
- **“Memory Mayhem”攻击（第5章）**：
  - **核心思想**：挑战“CPU 内部状态（寄存器、缓存）不受 Rowhammer 影响”的假设，证明在执行上下文切换或函数调用时，溢出（Spill）到进程栈中的寄存器值会进入 DRAM，从而变得脆弱。
  - **关键技术细节**：
    - **寄存器诱捕**：通过发送信号（SIGSTOP）或利用进程阻塞窗口，迫使目标进程将寄存器值压入栈中，并使用微架构竞争将其逐出到 DRAM。
    - **精确内存放置（Baiting）**：通过释放精心计算数量的“诱饵页”，在地址空间布局随机化（ASLR）存在的情况下，将目标栈变量精确放置到已知的、存在可翻转比特的物理页上。
    - **攻击实例**：通过翻转 SUDO 或 OpenSSH 中用于存储认证状态的栈变量（如 `matched` 或 `result`），绕过密码验证。
- **“LeapFrog”攻击（第6章）**：
  - **核心思想**：一种控制流劫持攻击，通过翻转栈中保存的返回地址（即程序计数器，PC），迫使进程跳过关键的安全检查指令，实现“指令跳步”。
  - **关键技术细节**：
    - **LeapFrog Gadget 发现**：开发了一个名为“MFS”的动态二进制插桩工具，基于 Intel Pin 框架自动搜索满足单比特翻转即可从安全验证代码跳转到绕过验证代码的指令地址对。
    - **栈指纹定位**：为克服 ASLR，识别栈中的不变值作为“指纹”，并通过与 PC 的相对偏移量确定攻击目标。
    - **攻击实例**：在 TLS 握手过程中，迫使客户端跳过服务器签名验证步骤，接受恶意服务器的签名；在 Rust 编译的代码中，绕过其内存安全检查，触发缓冲区下溢。

**层次三：仅服务层访问（无硬件共置，仅远程交互 API）**
- **“Super Suffixes”攻击（第7章）**：
  - **核心思想**：提出一种联合优化策略，生成同时绕过目标文本生成模型和其附带的安全防护模型（Guard Model）的对抗性后缀。
  - **关键技术细节**：
    - **两步生成**：首先，使用修改版 GCG 算法并针对“拒绝方向”生成能够绕过生成模型安全护栏的“主后缀”；然后，以此为基础，通过联合优化生成和防护模型的损失函数，生成“次后缀”，二者结合形成“超级后缀”。
    - **交替优化算法**：解决生成模型和防护模型分词器不同导致梯度无法直接共享的问题。算法交替进行线性近似：每隔 N 次迭代，切换优化目标（生成模型损失 vs. 防护模型损失），以此生成候选 token；随后通过完整前向传播，评估并选择能最小化加权联合损失（Ly_gen + Lguard）的候选。
    - **防御（DeltaGuard）**：作为对策，提出通过分类模型内部状态与“拒绝向量”的余弦相似度在整个 token 序列上的**动态变化轨迹**（时间序列），来检测对抗攻击，因为不同类别的输入（良性、恶意、带主后缀、带超级后缀）会呈现独特的变化模式（见图7.4）。

### 6. 论文的主要结论与发现

1.  **隔离假设的失效**：现代 AI 和云基础设施中，从硬件到服务的各个抽象层之间的隔离假设都是脆弱的，可以被跨层攻击利用。
2.  **CPU 内部状态不再安全**：寄存器值在上下文切换等操作期间会溢出到内存，从而使其暴露于 Rowhammer 等物理故障注入攻击之下，表明安全计算边界应延伸至处理器微架构外部。
3.  **内存安全的局限性**：即使是用 Rust 等高安全性语言编写的程序，其编译器强加的确定性安全检查也能被控制流劫持攻击（LeapFrog）绕过，因为这些攻击针对的是底层返回地址。
4.  **物理故障的新特性**：现代 DRAM 中，Rowhammer 引发的位翻转不是孤立的，而是呈现出“相邻比特翻转”和集群效应，这一物理特性可被用于更强大的密码分析（如 ECDSA 私钥恢复）和 AI 模型（如词表污染）攻击。
5.  **联合优化的威胁**：针对部署了多层安全防护（生成模型护栏 + 专用防护模型）的 AI 服务，攻击者可以通过联合优化策略生成“超级后缀”，同时绕过所有防护层，实现有效的越狱。
6.  **内在状态的防御潜力**：防御应转向模型内部。通过分析模型潜在表征与“拒绝方向”之间余弦相似度随时间变化的轨迹（DeltaGuard），可以有效检测出那些绕过表面级分类器的精心构造的对抗性提示。

---

### 7. 优点

- **系统性框架**：论文最大的亮点在于其层次化、系统化的研究框架，将看似不相关的物理、架构和算法攻击融合成一个完整的安全分析视图，为安全评估提供了新范式。
- **发现新颖且深刻**：多项成果打破了领域内的固有观念，如“寄存器安全”、“高安全性语言可防故障攻击”、“词表可被污染”，具有很高的学术价值和现实冲击力。
- **技术与工具创新**：开发了多种实用工具，如自动发现控制流劫持 Gadget 的 **MFS 仿真器**、对抗联合优化的 **DeltaGuard 防御方法**，以及利用页错误进行 **ASLR 部分反随机化**的技术。
- **理论与实践并重**：所有攻击均有在真实硬件和真实软件上的概念验证或端到端演示，而不仅是理论分析，并且提供了攻击所需的时间、成本等量化数据，说服力强。

---

### 8. 不足与局限

- **攻击假设要求较高**：多数硬件攻击（Spill The Beans, Rubber Mallet, Memory Mayhem, LeapFrog）要求攻击者与受害者在同一物理机上**共置**，这在某些云环境中虽可实现，但并非普遍默认存在的条件。Memory Mayhem 的攻击成功率受到诱饵页成功率的限制（约 30-40%），虽然指出可优化至 80%，但当前并非 100%。
- **人工干预与离线阶段**：部分硬件攻击（如 Memory Mayhem 和 LeapFrog 中的寄存器/PC 定位）需要复杂的离线剖析，甚至可能需要修改受害代码以获取唯一的“探针值”，这降低了攻击的即时性。
- **防御措施尚未深入验证**：如 DeltaGuard 等防御方法虽被提出，但其对更强适应性的、可能专门针对内部状态轨迹进行优化的二阶攻击的鲁棒性，尚未进行验证，仅作为未来工作的方向。
- **实验环境相对固定**：AI 攻击实验基于几款特定开源模型和防护模型，场景（如恶意代码生成）覆盖范围有限。对商业闭源、持续更新的 AI 服务攻击的泛化能力未作评估。
- **部分攻击停留在理论或模拟阶段**：如 Rubber Mallet 中的 ECDSA 私钥恢复攻击，论文主要进行了可行性的理论分析，完整的实证验证被作为未来工作。LeapFrog 的 MFS 工具检测 gadget 的效率还有提升空间。
