## 摘要
随着机密计算的日益普及，在机密虚拟机（CVMs）如AMD SEV-SNP中运行数据库已成为一种有吸引力的方法，能以最小的变化保护传统数据库管理系统的敏感云数据。然而，此类CVM中的分析查询通常会产生显著开销，而先前的数据库工作大多止步于对这些减速现象的基准测试，而非对其优化。我们表明，此问题源于硬件与软件的不匹配：查询优化器仍依赖基于KVM（非加密虚拟机）的成本假设，而这些假设在CVM中已不再适用。为解决此问题，我们提出了一种轻量级的CVM感知成本校准方法。它利用优化器已有的简单物理代理，对两个影响优化器的主要开销来源进行建模：数据移动和RMP相关转换。实验表明，该校准显著缩小了KVM/CVM的性能差距，恢复高达48%的性能，甚至在某些工作负载上超过了KVM基线。

## Abstract
With the growing adoption of Confidential Computing, running databases in confidential virtual machines (CVMs) such as AMD SEV-SNP has become an attractive way to protect sensitive cloud data with minimal changes to legacy DBMSs. However, analytical queries in such CVMs often suffer substantial overhead, and prior database work has largely stopped at benchmarking these slowdowns rather than optimizing them. We show that this problem stems from a hardware-software mismatch: query optimizers still rely on KVM-oriented (non-encrypted VM) cost assumptions that no longer hold in CVMs. To address this, we propose a lightweight CVM-aware cost calibration. It models two dominant sources of optimizer-facing overhead: data movement and RMP-related translation using simple physical proxies already available to the optimizer. Experiments show that the calibration significantly narrows the KVM/CVM performance gap, recovering up to 48 percent performance and even outperforming the KVM baseline on some workloads.

### 2. 论文提出的方法论

*   **核心思想**：
    *   在不改变优化器架构的前提下，利用优化器已有的统计信息（如页面数、元组数、内存使用量），对原有的操作符成本模型中引入针对性的惩罚项，以反映 RMP 敏感型随机访问和数据移动的开销。
*   **关键技术细节**：
    1.  **核心概念：工作集（Working Set, \( ws \)）与溢出因子（Overflow Factor, \( x \)）：**
        *   \( ws \)：操作符执行期间必须保持活跃的数据量，从优化器可见的层面级（页数、元组数）近似得到。
        *   \( x = \max(0, (ws - (1 + \epsilon)C) / C) \)：当工作集超过有效缓存容量 \( C \) 时激活，用于量化超出缓存的数据量，\( \epsilon \) 是容忍因子，防止估算误差造成不稳定性。
    2.  **两类核心开销的建模公式：**
        *   **数据移动开销**：`datamovecost(ws, x) = ws · η_dm · σ(x)`
            *   当工作集大于缓存时触发（通过激函数 \( \sigma(x) \)），与工作集大小成正比，`η_dm` 是标定后的单位惩罚系数。
        *   **RMP 相关转换开销**：`rmptcost(a, x) = a · η_rmp · (1 + α_rmp · σ(x))`
            *   `a` 代表页面访问强度，`η_rmp` 是每次访问的惩罚系数，`α_rmp` 在内存溢出时提高代价，反映了地址转换和完整性校验带来的时延放大。
    3.  **为不同操作符定制成本模型**：
        *   **扫描操作符（顺序扫描/位图扫描）**：主要对数据移动敏感，因此在基础成本上增加 `datamovecost`。
        *   **索引扫描**：对指针追踪和 RMP 检查敏感，因此增加 `rmptcost`，其访问强度参数 `a` 由“预计返回行数 × B+树高度”来估算。
        *   **连接操作符**：
            *   **嵌套循环连接（NL）**：既可能因重复索引探测产生 RMP 开销，也可能因数据量大产生数据移动开销，故在特定条件下同时增加两项。
            *   **哈希连接（HJ）**：对工作集大小极为敏感。当构建的哈希表超过末级缓存（LLC）阈值时，会同时增加数据移动和 RMP 开销。
            *   **归并连接（MJ）**：主要对流式 I/O 和排序敏感，因此以数据移动开销为主，辅以条件性 RMP 开销。
        *   **物化/记忆操作符（MAT/MEM）**：因其主要保存中间状态，对工作集大小敏感，因此增加 `datamovecost`。
*   **参数标定**：
    *   轻量级的调参过程，使用每个工作负载中随机抽取的三分之一查询进行代表性子集调优，目标是找到一组在所有工作负载上都表现稳健的参数。
