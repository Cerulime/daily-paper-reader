## 摘要
符号执行是一种强大的程序分析技术，广泛用于漏洞检测、安全测试和恶意软件分析等应用。然而，由于真实程序通常存在某些结构和语义模式，该技术面临可扩展性问题，例如路径爆炸和复杂约束。现有方法尝试通过将程序转换为新的表示来规避这些模式，以降低执行开销。遗憾的是，这些转换往往过于僵化，无法利用多样的局部程序语义，有时还依赖于为具体执行而设计的编译器优化，这可能与符号执行的目标不符。我们提出Symbolon，这是一个自动学习多样代码转换并根据上下文应用它们以改进符号执行的框架。我们的关键洞见是将转换发现定义为一个在程序表示上的搜索问题。为使搜索实用化，Symbolon以低成本方式在小程序上离线学习转换，将其提炼成可复用的代理技能库，并使用一个代理在仓库级目标上实例化这些技能。我们的评估表明，Symbolon在32个真实程序的16种搜索策略上显著改进了符号执行引擎KLEE，将行覆盖率平均提升了3.69倍，同时将峰值内存和每次查询求解器时间分别降低了29.2倍和123倍。当应用于最新Linux内核时，Symbolon发现了21个此前未知的错误，所有这些错误均已报告给内核维护人员。

## Abstract
Symbolic execution is a powerful program analysis technique with broad applications, such as vulnerability detection, security testing, and malware analysis. However, this technique is known to suffer from scalability issues, e.g., path explosion, complex constraints, due to certain structural and semantic patterns commonly presented in real-world programs. Existing approaches attempt to escape these patterns by transforming programs into new representations to reduce the execution cost. Unfortunately, these transformations are often too rigid to exploit diverse local program semantics and sometimes rely on compiler optimizations designed for concrete execution that may misalign with the goals of symbolic execution.   We present Symbolon, a framework that automatically learns diverse code transformations and applies them context-sensitively to improve symbolic execution. Our key insight is to formulate transformation discovery as a search problem over program representations. To make the search practical, Symbolon learns transformations cheaply offline on small programs, distills them into a reusable library of agent skills, and uses an agent to instantiate these skills on repo-level targets. Our evaluation shows that Symbolon substantially improves the symbolic execution engine KLEE across 16 search strategies on 32 real-world programs, increasing line coverage by 3.69x on average while reducing peak memory and per-query solver time by 29.2x and 123x, respectively. When applied to the latest Linux kernel, Symbolon uncovers 21 previously unknown bugs, all of which have been reported to the kernel maintainers.

## 1. 论文的核心问题与整体含义

- **研究背景**：符号执行是一种广泛应用于漏洞检测、安全测试与恶意软件分析的程序分析技术。它在探索程序路径时会系统地累积约束，但面临严重的可扩展性问题，主要表现为路径爆炸和复杂约束，这些通常源于程序中常见的结构模式（如符号循环边界、复杂数据依赖、符号内存分配、无用的 I/O 操作等）。

- **核心问题**：程序的表示方式（代码结构）对符号执行成本具有高度敏感性。等价的程序实现可能表现出截然不同的分支复杂度、内存访存模式和约束求解难度。现有的代码转换方法（如编译器优化）通常是固定的，且面向具体执行目标设计，与符号执行的目标（如提高覆盖率、减少求解器开销）不一致。人工编写的转换规则过于僵化，无法在多样的局部上下文中自适应地应用。

- **整体含义**：论文提出一种全新的范式——通过自动搜索和学习程序表示层面的转换，来系统性地改善符号执行的可扩展性，而非仅依赖引擎内部优化或固定的变换规则。

## 2. 论文提出的方法论

### 核心思想

- 将“转换发现”定义为在程序表示空间中的**搜索问题**：给定一个程序，寻找一个源级代码表示的变体，使得对该变体执行符号执行后，生成的测试回放到原始程序上能获得更优的指标（如行覆盖率）。
- 关键设计在于使用**转换后回放（transform-and-replay reward）**：转换后的程序仅作为测试生成的“脚手架”，其正确性不要求与原程序语义等价；只要求生成的测试在**原始程序**上回放时能够改进目标指标。这允许探索语义破坏的转换，同时保证优化目标与原始程序上的符号执行目标完全对齐。

### 两阶段架构

#### 阶段一：离线学习（Offline Learning）

在小规模程序上进行，以避免高昂的符号执行与回放成本。包含三个步骤：

- **符号执行剖析（Profiling）**：对原始程序运行符号执行，收集执行轨迹（分支、约束、求解查询、状态终止原因等），并使用大语言模型（LLM）从代码和轨迹中提出**候选障碍（barriers）**——即阻碍符号执行的代码模式假设。

- **转换搜索（Transformation Search）**：针对每个候选障碍，利用 LLM 引导的演化搜索在程序表示空间中寻找有效的转换。每一轮由 LLM 提议一个转换后的程序，编译后运行 KLEE 生成测试，再回放到原始程序上测量覆盖率。只有回放覆盖率提升的转换才被保留为验证样例。搜索预算为每个程序 20 次迭代，且很多程序在 14 次迭代内就达到 100% 回放覆盖率。

- **技能提炼（Rule Learning / Skill Distillation）**：对通过验证的转换样例进行泛化，生成额外的上下文变体，再经回放验证后，归纳为可复用的**代理技能**（skill）。每个技能包含：障碍描述、适用条件、转换步骤、以及带有回放证据的 before/after 示例。最终得到一个包含 211 个技能的库，这些技能可持久化、可检查，无需更新模型权重。

#### 阶段二：在线推理——仓库级转换（Repo-Level Transformation）

- 在真实大型项目上应用时，代理（Agent）利用上述技能库，以**上下文敏感**的方式在目标代码中定位候选位置，并实例化技能。代理可以观察到局部源码上下文（变量名、类型、注释、控制流），并获得来自编译、符号执行轨迹和回放覆盖率的多轮反馈，从而过滤无效或不适当的转换。

- 算法流程（简化描述）：
  1. 从目标项目开始，先在原始项目上评估一次获得基线反馈和覆盖率。
  2. 每一轮，代理根据当前代码、技能库和上一轮反馈生成新的转换候选。
  3. 对候选进行编译检查；若编译失败则记录诊断信息并跳过。
  4. 若编译成功，则在转换后的代码上运行符号执行并回放测试到原始项目；若回放指标提升，则更新所选的最佳脚手架。
  5. 在固定轮数后输出最终的脚手架，用于后续正式的符号执行。

- 转换通过 `#ifdef KLEE_BITCODE` 隔离，确保同一个源码树既能产生用于 KLEE 的转换后 bitcode，又能构建未经修改的原生版本用于覆盖率回放。

## 6. 论文的主要结论与发现

1. **覆盖率的显著提升**：在 32 个程序上，Symbolon 使 KLEE 的行覆盖率平均提高了 **3.69 倍**（按程序汇总）和 **3.87 倍**（按搜索策略汇总）。30 个程序获得提升，其中 19 个程序在所有搜索策略下均超越基线。

2. **搜索策略依赖降低**：原始程序上最强策略与最弱策略的覆盖率差距为 7.4 倍，应用 Symbolon 后这一差距降至 1.7 倍，说明转换后的代码对各种探索策略都更友好。

3. **执行效率大幅改善**：峰值内存平均降低 **29.2 倍**，每次查询平均求解时间降低 **123 倍**，大多数程序的内存和求解开销都显著减少。

4. **编译器优化的补充**：Symbolon 能够在现有编译器优化标志（包括 `-O2`、`-sccp` 等）的基础上将覆盖率平均再提升 **3.52 倍**，证明其与刚性优化互补。

5. **真实安全漏洞发现**：与基线相比，Symbolon 发现了额外 240 个 UBSan 违规和 25 个 ASan 违规。结合 SyzSpec 与 syzkaller，在内核模糊测试中将总边缘覆盖率提升 2.47 倍，并发现了 **21 个此前未知的 Linux 内核错误**（use-after-free、越界访问、空指针解引用等），已全部报告。

6. **泛化能力**：在 Rust、Python 和 JavaScript 的符号执行引擎上，Symbolon 均能直接带来覆盖率提升（+56% 汇总），揭示了源级转换知识的跨语言可迁移性。

7. **成本可控**：离线学习一次摊销，在线部署每项目平均 21.44 分钟和 $11.58 的费用，在 12 小时预算中占比很小（约 3%）。

## 7. 优点

- **范式创新**：将代码转换从手工编写或编译器固定优化升级为自动搜索与学习问题，开创了“学习程序表示以优化分析”的方向。
- **目标对齐巧妙**：transform-and-replay 奖励机制允许探索语义破坏转换，同时始终以原始程序上的表现作为优化目标，解决了转换目标与分析目标错位的问题。
- **技能知识复用**：离线学习、蒸馏为可持久化技能库的设计，使得昂贵的搜索仅在简化的小程序上进行，而知识可迁移至大型真实项目，且无需模型微调，易于审计和更新。
- **上下文敏感性**：代理利用源码上下文和多轮反馈实例化技能，避免全局统一的僵化应用，能够根据局部语义（如变量使用、后续代码依赖）自适应调整。
- **集成无关性**：转换发生在源级，与具体的符号执行引擎内部策略或求解器无耦合，因此能一致地提升所有 16 种搜索策略的性能。
- **实验全面扎实**：覆盖大量程序、策略和消融维度，并包含实际漏洞发现与报告，学术价值和应用价值兼具。
- **开放与可重现**：作者公开了技能库、代理框架和转换后的项目代码。

## 8. 不足与局限

- **学习-部署差距**：技能库从小型竞赛程序学习，可能缺失大型项目的领域特定 API、宏定义和长距离依赖；一些真实障碍可能未被覆盖。
- **部分项目无改善**：cjson 和 nasm 的平均覆盖率未见提升（虽某些策略下有效），说明当前技能库或搜索机制仍无法处理所有分析障碍。
- **LLM 依赖与成本不确定**：转换质量和最终效果对 LLM 能力敏感，不同模型间表现差异可达 23.5%（如使用 GPT 5.5 高努力模式与默认相比），且 API 调用存在持续费用。
- **部署时间仍占预算**：虽然目前部署时间仅占约 3%，但在某些即时性要求高的场景或极短时间预算下，转换开销仍可能成为负担。
- **泛化实验较初步**：跨语言/跨引擎实验仅覆盖少量项目和默认配置，未深入调整技能库或针对新引擎进行重新学习，结果可能不完全反映真实潜力。
- **未处理所有符号执行限制**：方法主要聚焦于程序表示层面，对引擎内在约束（如求解器能力不足、状态合并策略）未直接改善；符号大小分配等问题也仅是绕过而非从根本上解决。
- **回放依赖**：强化奖励完全依赖在原始程序上回放测试，如果生成的测试因非确定性或环境差异无法正确回放，可能漏掉有价值的转换。
