## 摘要
云GPU租户会获得一个型号名称和区域，但无法直接检查运行其工作的物理加速器。我们为此场景提出了一种纯软件认证原语。一个CUDA探针使用物理SM标签和依赖全局加载来测量SM按内存区域的延迟矩阵。一个流式归约器将充分统计量、配置、代码哈希、网络证据和压缩的原始数据归档提交到证书中，验证者无需GPU即可检查。该证书支持三种声明。首先，每个SM的延迟映射是一个稳定的物理指纹。在RTX 5090满负载六小时的运行中，其中位时间抖动为0.09个周期，而仅凭形状留一法分类能以100.0%的准确率区分不同的Blackwell芯片。其次，绕过缓存的HBM扫描可跨代恢复硬件类拓扑，包括统一的Volta V100内存域、双向Hopper H200 L2拆分，以及Blackwell B200双芯片NV-HBI封装，其74/74 SM分区带有30个周期、15.5纳秒的跨芯片代价。第三，公共网络地标将同一证书绑定到粗略位置。在B200运行中，169个RIPE Atlas探针将服务器定位在其声称的数据中心44公里范围内，并拒绝了所有11个诱饵站点。综合这些测量，无需特权访问或供应商密钥即可检查云GPU的身份、类别和粗略位置。

## Abstract
Cloud GPU tenants receive a model name and a region, but cannot directly inspect the physical accelerator that runs their job. We present a software-only attestation primitive for this setting. A CUDA probe measures an SM-by-memory-region latency matrix using physical SM labels and dependent global loads. A streaming reducer commits sufficient statistics, configuration, code hashes, network evidence, and a compressed raw data archive into a certificate that a verifier can check without a GPU. The certificate supports three claims. First, the per-SM latency map is a stable physical fingerprint. Over a six-hour full-load RTX 5090 run, its median temporal jitter is 0.09 cycles, while shape-only leave-one-out classification separates distinct Blackwell dies with 100.0% accuracy. Second, cache-bypassing HBM sweeps recover hardware-class topology across generations, including a unified Volta V100 memory domain, a two-way Hopper H200 L2 split, and a Blackwell B200 two-die NV-HBI package whose 74/74 SM partition carries a 30-cycle, 15.5 ns cross-die penalty. Third, public network landmarks bind the same certificate to a coarse location. In the B200 run, 169 RIPE Atlas probes place the server within 44 km of its claimed datacentre and reject all 11 decoy sites. Together, these measurements check cloud-GPU identity, class, and coarse location without privileged access or a vendor key.

- **核心问题**：云GPU租户仅能获得供应商宣称的型号、区域和价格，却无法直接验证实际的物理加速器是否为所租用的那一颗、是否被掉包、是否位于合同约定的地理位置。现有的可信硬件（如TEE）认证依赖特权访问或厂商密钥，但大量消费级和工作站级GPU并不提供这种证明路径。

### 2. 方法论
- **核心思想**：将GPU的物理芯片指纹（由片上互连延迟决定的每SM延迟映射）、内存拓扑特征（跨缓存一致性与分区形状）以及网络地标距离绑定为一条不可伪造的证书链，供第三方无GPU离线验证。
- **关键技术细节**：
  - **延迟探针**：利用PTX特殊寄存器`%smid`读取物理SM ID，并通过动态共享内存迫使每个SM仅驻留一个block；线程0使用自旋锁互斥进入计时区，预热依赖指针链后执行一系列带`.cg`属性的全局加载（`ld.global.cg.u32`），以绕过L1缓存，测量到L2/全局内存的延迟。
  - **认证流水线**：原始流`R`（含SM、区域、周期等元组）经由流式归约器生成有界充分统计摘要`S`（计数、均值、方差、极值等）。证书`C = H(source) ∥ H(config) ∥ H(S) ∥ H(Rarchive) ∥ M`，其中`M`包含决策规则与设备元数据。验证者仅需重算哈希，无需GPU或完整原始数据。
  - **三大认证声明**：
    1. **物理身份**：利用SM×探针区域的延迟矩阵作为稳定指纹，在重负载下依然保持极低时间抖动。
    2. **硬件类别**：通过绕过缓存的HBM扫描（`ld.global.cv`），恢复内存拓扑的分区特征（统一L2、双路L2分片或双芯片NV-HBI拆分），并与宣称的架构类对比。

### 7. 优点
- **纯软件、无特权**：不需要TEE、固件接口或硬件密钥，适用于大量消费级和工作站级GPU。
- **多维证明**：同时认证物理身份、硬件类别和地理位置，构成较为完整的云GPU信任链。
- **离线验证友好**：验证者仅需摘要和哈希值，无需GPU或原始数据，显著降低验证门槛。
- **高稳定性**：在满负载和高内存占用下指纹依然稳定，表明其对抗动态环境噪声的鲁棒性。
- **结合网络证据**：有效利用第三方公网测度（RIPE Atlas）实施距离限制，并通过指纹绑定防止代理重放攻击。
- **良好的可复现封装**：arXiv源包中包含压缩的原始数据、CUDA代码、归约器、验证器及清单，保证实验可审计。

- **定位精度粗**：定位仅达大都市或洲际尺度，无法精确到机架；且依赖实例开放TCP端口进行探测。
