## 摘要
定向模糊测试引导模糊器走向用户定义的汇点函数以识别漏洞，但即使经过长时间测试也常常无法触发崩溃。我们识别出两个阻碍定向模糊器暴露崩溃的挑战：对间接调用的不完整静态分析，使得可达路径对基于距离的指导不可见；以及缺乏针对崩溃前提的语义指导，导致在实用时间预算内盲目变异无法满足这些前提。一个自然的干预点是初始种子语料库：编码了正确控制流路径并满足关键崩溃前提的种子，将模糊测试从盲目探索转变为局部细化。现有的种子生成方法均未解决这两点：基于语法和格式驱动的方法生成结构有效的输入，但缺乏汇点感知；基于LLM的方法要么缺乏汇点定位，要么通过单次提示继承了静态分析的局限性。我们提出SeedSmith，一个模仿安全分析师工作流程的智能LLM管道：从汇点开始，迭代探索代码库，解析间接调用，识别崩溃前提，并合成满足这些前提的具体输入。由于SeedSmith作为种子生成前端运行，其种子与模糊器无关，无需修改即可提升任何下游基于变异的模糊器。在Magma上，使用SeedSmith种子的模糊器相比默认种子实现了几何平均崩溃时间加速11.51倍（AFL++）至14.66倍（AFLGo）。在ARVO上，SeedSmith使模糊器触发了16个之前无法到达的bug，涉及10个具有不同输入格式的项目。

## Abstract
Directed fuzzing steers fuzzers toward user-defined sink functions to identify vulnerabilities, but it frequently fails to trigger crashes even after long campaigns. We identify two challenges that prevent directed fuzzers from exposing crashes: incomplete static analysis of indirect calls, which leaves reachable paths invisible to distance-based guidance, and lack of semantic guidance for crash preconditions, which blind mutation cannot satisfy within practical time budgets. A natural intervention point is the initial seed corpus: seeds that encode the right control-flow path and satisfy key crash preconditions shift fuzzing from blind exploration to local refinement. Existing seed generation approaches address neither: grammar-based and format-driven methods produce structurally valid inputs with no sink awareness, while LLM-based methods either lack sink targeting or inherit static analysis limitations through one-shot prompting. We present SeedSmith, an agentic LLM pipeline that replicates a security analyst's workflow: starting from a sink, it iteratively explores the codebase, resolves indirect calls, identifies crash preconditions, and synthesizes concrete inputs that satisfy them. Because SeedSmith operates as a seed generation front-end, its seeds are fuzzer-agnostic and improve any downstream mutation-based fuzzer without modification. On Magma, fuzzers using SeedSmith seeds achieve geometric mean crash-time speedups of 11.51 times (AFL++) to 14.66 times (AFLGo) over default seeds. On ARVO, SeedSmith enables fuzzers to trigger 16 previously unreachable bugs spanning 10 projects with diverse input formats.

### 论文核心问题与整体含义（研究动机和背景）
定向模糊测试旨在将模糊器引导至指定的“汇点函数”（Sink Functions）以发现特定漏洞，但其有效性常受两大核心挑战制约：
*   **挑战一（C1）**：静态分析无法完全解析间接调用（如函数指针、虚函数调度），导致基于距离的调度算法无法看到完整的可达路径，使模糊器失去有效的方向指引。
*   **挑战二（C2）**：即使模糊器抵达汇点，触发崩溃往往需满足复杂的输入前置条件（如特定字段值、数据流依赖），而传统的盲目字节级变异在有限时间内难以同时满足所有条件。

现有种子生成技术未能有效解决上述挑战。语法或格式驱动的方法缺乏对特定汇点的感知；而基于大语言模型（LLM）的方法依赖于静态分析结果或单次提示，存在局限性。本研究旨在通过一种智能的、迭代式探索的工作流，生成能够编码正确控制流路径并满足崩溃前置条件的高质量种子，从根本上解决定向模糊测试的“冷启动”难题。

### 论文提出的方法论
SeedSmith 是一个“智能体驱动”（Agentic）的LLM流水线，模拟人类安全分析师的工作流程。其核心思想是作为模糊器无关的前端生成种子，整个系统由三个阶段构成：

*   **阶段一：目标准备**
    1.  **项目索引**：提取项目中所有函数及其边界，构建可查询的知识库。
    2.  **调用路径识别**：利用 CodeQL 抽取静态调用图，并对间接调用进行类型匹配解析。通过“路径压缩”算法，将起始函数（Harness）到汇点的多分支路径合并为一条紧凑的线性化路径，供下游LLM智能体参考。
    3.  **项目构建**：使用 Sanitizer（如ASAN、MSAN）编译目标程序，用于种子验证。

*   **阶段二：智能代码探索（分析智能体）**
    1.  输入起始函数代码、汇点函数代码及压缩后的静态调用路径（可能为空）。
    2.  配备**上下文感知的代码搜索工具**：智能体可迭代搜索代码库，工具返回语义完整的代码单元（如函数体、类型定义等），而非简单的文本行片段。
    3.  LLM通过多步推理，主动恢复静态分析遗漏的间接调用路径（解决C1），并提取触发崩溃所需的数据流依赖和条件检查（解决C2），最终生成一份详尽的结构化**分析报告**。

*   **阶段三：智能种子生成（种子生成智能体）**
    1.  消费分析报告，利用轻量级LLM生成Python脚本来程序化构造测试输入，而非直接输出原始二进制，以处理复杂文件格式的结构限制。
    2.  **迭代式精炼循环**：执行脚本生成种子，并在Sanitizer插桩的二进制上测试。若未崩溃，则收集执行反馈（如报错信息）传递给LLM以修正脚本，最多循环10轮。循环终止时收集所有生成的种子作为最终语料库。

*   **挑战解决实证**：通过案例分析（如nginx and openjpeg），详细展示了SeedSmith如何通过LLM的代码理解能力，恢复间接调用路径并构造满足复杂结构约束的恶意输入。

### 优点
*   **模糊器无关的即插即用**：作为前端种子生成器，无需修改任何下游模糊器内部逻辑即可带来定向测试能力的显著提升。
*   **创新的智能体架构**：首次将迭代式、工具辅助的LLM智能体流程应用于种子生成，有效模拟了安全专家的推理路径，解决了静态分析的根本性缺陷。
*   **轻量级与高性价比**：种子生成耗时短、成本低，却能带来数倍乃至数十倍的模糊测试效率提升。
*   **精细化的消融分析**：清晰验证了上下文感知搜索和调用图支持两个核心设计的有效性及其对不同类型模糊器的影响。

### 不足与局限
*   **对LLM能力的依赖性**：系统的核心分析能力高度依赖底层LLM对代码和漏洞的理解，若LLM对特定漏洞推理失误，将会导致种子生成失败。
*   **汇点函数的精确性**：当前实验中的汇点函数是已知的崩溃点入口，但真实漏洞的根因可能不在给定函数内。若汇点定位不准，系统可能无法定位真正的触发条件。
*   **零日漏洞发现的局限性**：评估集中于N-day漏洞复现，SeedSmith本身不具备自动定位汇点的能力，用于零日发现时需与其他漏洞定位组件结合。
*   **语言与漏洞类型的广度**：实验主要在C/C++项目上验证，其对其他编程语言（如Java, Rust）及更复杂的逻辑漏洞的有效性有待进一步探索。
