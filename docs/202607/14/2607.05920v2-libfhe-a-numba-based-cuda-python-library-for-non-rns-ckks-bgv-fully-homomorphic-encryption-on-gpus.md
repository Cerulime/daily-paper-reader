## 摘要
自第四代全同态加密框架CKKS提出已过去十年，然而至今仍未有迹象表明会出现第五代后继者；近年来，许多研究探索了GPU加速以提升同态计算的效率。本文提出LibFHE，一个高性能GPU加速框架，其特色在于采用CUDA-Python绑定，为同态工作负载同时实现高级可编程性与裸机级GPU性能。当前最先进的实现大多采用RNS-CKKS变体。与此不同，本工作特意重新审视了原始（非RNS）CKKS-BGV框架，并开发了基于GPU的实现及相应优化。实验结果表明，优化的CUDA-Python实现可以达到与高度优化的基于CPU的C++同态加密库相当的性能，同时显著降低实现复杂度并提升可编程性。

## Abstract
It has been a decade since the fourth-generation FHE framework, CKKS, was proposed; yet, there is still no indicator pointing toward a fifth-generation successor; and in recent years, numerous studies have explored GPU acceleration to improve the efficiency of homomorphic computations. In this paper, we propose LibFHE, a high-performance GPU-accelerated framework that features CUDA-Python bindings to achieve both high-level programmability and bare-metal GPU performance for homomorphic workloads. A large majority of state-of-the-art implementations adopt the RNS-CKKS variant. In contrast, this work deliberately revisits the original (non-RNS) CKKS-BGV framework, and develops a GPU-based implementation along with corresponding optimizations. Experimental results demonstrate that optimized CUDA-Python implementations can achieve performance comparable to highly optimized CPU-based C++ FHE libraries, while significantly reducing implementation complexity and improving programmability.

### 2. 论文提出的方法论

*   **核心思想**：
    1.  **技术路线重定向**：放弃主流极度优化的全 RNS-CKKS 方案，回归原始的、非 RNS 的 CKKS 和 BGV 方案。这样做的目的是**规避 RNS 所带来的模数链刚性、复杂的密钥切换和编程不灵活性**。
    2.  **高级语言实现**：完全使用 Python 和 Numba（一个 JIT 编译器）构建，利用 NVIDIA 的 `cuda-python` 低层级绑定，实现对 GPU 的裸机级控制，从而弥合高生产力与高性能之间的鸿沟。

*   **关键技术细节与算法流程**:
    *   **混合大整数表示策略**：
        *   **基本表示**：采用基于 32 位肢的**多精度基数表示**作为主要数据组织格式。这为动态精度控制和缩放提供了灵活性。
        *   **按需转换**：仅在执行计算量最大的**多项式环乘法**时，才将系数从基数表示**原地转换**为 RNS 表示。
        *   **乘法流水线**：`基数肢表示` -> `RNS 分解` -> `扭曲因子` -> `正向负循环 NTT` -> `逐点乘法` -> `逆向 NTT` -> `逆扭曲` -> `RNS 域` -> `通过 Garner 算法 (MRC) 重构回基数肢表示`。这个过程充分利用了 NTT 的 O(N log N) 复杂度和 RNS 的并行性。
    *   **GPU 专用优化**：
        *   **基于矩阵的四步 NTT**：为了解决 NTT 中蝴蝶操作不规则的内存访问模式，将一维多项式重塑为二维矩阵。通过**行变换 -> 矩阵转置 -> 行变换**的方式，将跨步访存转化为连续访存，并采用**分块（Tiling）** 和**分层内核策略**以更好地利用共享内存和减少全局内存访问。
        *   **预计算**：将所有 NTT 扭转因子、模逆元、RNS 转换常数等在 CPU 端预计算好并传输到 GPU 显存，以最小化实时计算开销。
        *   **操作抽象 (CipherTensor)**：引入 `CipherTensor` 数据结构，将多个具有相同属性的密文打包成一个张量。通过 `cnum` 批处理维度，支持**成对（pairwise）** 和**广播（broadcast）** 两种执行模式，实现 SIMD 风格的并行处理，最大化 GPU 占用率。
        *   **Karatsuba 乘法**：在密文乘法中，利用 Karatsuba 算法减少所需的基础多项式乘法次数。
