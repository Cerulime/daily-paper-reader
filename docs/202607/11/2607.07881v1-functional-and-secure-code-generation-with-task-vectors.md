## 摘要
大型语言模型（LLMs）越来越多地用于代码生成，但它们难以生成既功能完备又无安全漏洞的代码。先前提升此类编码LLM安全代码生成能力的工作大多侧重于使用不同数据集分别评估代码功能与安全性，或侧重于在生成后查找漏洞。与此同时，文本生成领域在对齐技术上取得了显著进展，通过对模型进行调优，使其输出展现特定品质（如有用性、无害性）。特别值得关注的是任务向量算术，通过对LLM权重进行线性运算，可以任意增强对齐效果，且仅带来极小的计算开销。我们开发了一种新方法SecVecCoder，利用任务向量生成可信赖的代码，同时兼顾功能性与安全性，且无需生成后调整。在CodeGuard+基准上对来自三个家族的六种编码LLM进行测试，SecVecCoder将可信赖代码补全率较基础模型提升了2.1至36.0个百分点，对未见过的CWE类型的改进幅度最高可达39.1个百分点。由于编码LLM的有效性仅依赖于改变模型权重，SecVecCoder无需针对特定方法的解码，因此解码延迟平均仅比基础模型高出0.6%以内。

## Abstract
Large language models (LLMs) are increasingly used for code generation, but they struggle to generate functional code free of security vulnerabilities. Prior work to improve the secure code generation abilities of such coding LLMs has largely focused on evaluating code functionality and security separately using different datasets, or focused on finding vulnerabilities post-generation. At the same time, the text-generation domain has seen significant work on alignment techniques, where models are tuned such that their outputs exhibit certain qualities (e.g., helpfulness, harmlessness). Of particular interest is task-vector arithmetic, where linear operations on LLM weights can be used to arbitrarily enhance alignment while incurring only minimal computational overhead. We develop a novel method, SecVecCoder, leveraging task vectors to produce trustworthy code that is simultaneously functional and secure without the need for post-generation adjustment. Across six coding LLMs from three families on the CodeGuard+ benchmark, SecVecCoder improves the rate of trustworthy code completions by 2.1-36.0 percentage points over the base model, with improvements on unseen CWE types reaching up to 39.1 percentage points. Since the effectiveness of the coding LLM relies only on changing the model weights, SecVecCoder requires no method-specific decoding and hence achieves a decoding latency within 0.6% of the base model's, on average.

- **问题背景**：大型语言模型（LLM）在代码生成中虽表现出色，但往往难以兼顾**功能正确性**与**安全性**，所生成代码常含有可利用的安全漏洞。  
- **现有局限**：  
  - 多数已有工作**分离评估**功能和安全性（如在不同数据集上分别测试），无法直接衡量同一段代码是否“既功能正常又安全”。  
  - 已提出的训练时方法（如安全指令微调、前缀调优）需要大量数据和计算；推理时方法（如约束解码、辅助安全模块）会显著增加解码开销或需要额外训练组件。  
- **整体含义**：本文探索一种**轻量且可调制的可信代码生成方法**，借鉴LLM对齐中的“任务向量算术”（task-vector arithmetic），使模型**在不改变解码过程的情况下**，通过直接调整模型权重，同时提升生成代码的功能性与安全性。

## 2. 论文提出的方法论  

- **核心思想**：  
  - 将代码生成的安全性视为一种行为对齐问题，通过**任务向量的线性运算**将安全行为“注入”基础模型权重。  
- **技术细节与公式**：  
  1. **构建任务向量**：  
     - 记基础模型权重为 \(W_{\text{base}}\)。  
     - 分别用安全代码和不安全代码微调，得到 \(W_{\text{sec}}\) 和 \(W_{\text{vul}}\)。  
     - 定义安全任务向量 \(\tau_{\text{sec}} = W_{\text{sec}} - W_{\text{base}}\)，不安全任务向量 \(\tau_{\text{insec}} = W_{\text{vul}} - W_{\text{base}}\)。  
  2. **任务向量算术操作**（最终模型权重 \(W_{\text{steered}}\)）：  
     \[
     W_{\text{steered}} = W_{\text{base}} + \alpha \tau_{\text{sec}} - \theta \tau_{\text{insec}}
     \]
     - 研究三种 **操作模式**：  
       - **Negation**（\(\alpha = 0\)）：\(W_{\text{base}} - \theta \tau_{\text{insec}}\)（仅压制不安全方向）。  
       - **Contrast**（\(\alpha = \theta\)）：\(W_{\text{base}} + \theta(\tau_{\text{sec}} - \tau_{\text{insec}})\)（沿安全-不安全差异方向移动）。  
       - **Secure-Anchored**（\(\alpha = 1\)）：\(W_{\text{base}} + \tau_{\text{sec}} - \theta \tau_{\text{insec}}\)（以安全微调模型为锚点，再减去不安全方向）。  
  3. **微调策略选择**：  
     - 比较了**SFT**、**SafeCoder风格**和**局部偏好优化（LPO）** 三种微调方法提取任务向量的效果。  
     - 通过余弦相似度和符号一致率发现，LPO得到的 \(\tau_{\text{sec}}\) 与 \(\tau_{\text{insec}}\) 负相关程度最高，方向区分最明显，表明LPO能更好地分离安全与不安全行为。  
     - 最终方法 **SecVecCoder** 采用**LPO衍生任务向量 + Secure-Anchored操作**，默认强度 \(\theta=1.5\)。  
  4. **训练细节**：  
     - 使用LoRA（秩16）进行参数高效微调，LPO适配器从SFT适配器初始化，训练3-5个epoch。  
     - 损失函数强调安全/不安全代码差异位置（局部化偏好优化）。

- **有效性**：  
  - LPO衍生任务向量配合Secure-Anchored操作在 \(\theta=1.5\) 时，在6个模型上均**稳定提升 sec-pass@1**，相对基础模型提升 **22.9%–61.5%**，平均**36.1%**。

  - 对**未见CWE类型**的泛化性良好，相对基础模型绝对提升最高 **39.1 个百分点**。  
- **效率与成本**：  
  - 训练成本仅为SafeCoder的 **8.0%–12.5%**，SCoDE的 **28.7%–37.9%**。  
  - 推理延迟与基础模型几乎无异（平均+0.6%），远优于需要特定解码的方法（CoSec、SCoDE、DeepGuard）。  
- **关键设计验证**：  
  - LPO是构造有效任务向量的关键，能使安全/不安全方向明显分离。  
  - Secure-Anchored操作在多数情况下优于Negation和Contrast。  
  - 两个向量分量（\(\tau_{\text{sec}}\) 和 \(\tau_{\text{insec}}）均不可或缺，移除任一项都会导致 sec-pass@1 显著下降。

## 7. 优点  

- **方法新颖**：首次将任务向量算术应用于可信代码生成，无需额外解码组件，直接通过权重调整实现行为对齐。  
- **轻量高效**：训练成本极低，推理零额外开销，实际部署友好。  
- **可调制性强**：通过简单的缩放系数 \(\theta\) 即可调整安全行为强度，无需重新训练。  
- **评估严格统一**：紧扣“同时功能且安全”的联合评价标准，使用CodeGuard+在同一完成上衡量两项属性，纠正了以往分离评估的偏差。  
- **分析全面**：细致比较了微调策略、算数操作、强度选择、泛化性与消融研究，结论可信度高。

## 8. 不足与局限  

- **个别模型表现不最优**：在DeepSeek-Coder-6.7B上，sec-pass@1（68.2%）落后于SafeCoder（79.3%）和SVEN（75.5%），最佳操作强度可能需要进一步调节。

  - 训练仅使用来自SVEN的803对样本，规模较小，且仅覆盖9类CWE。对于更广泛的安全弱点类型，效果尚待验证。
