## 摘要
大语言模型智能体正越来越多地应用于漏洞分析，但现有基准测试已跟不上发展。它们通常依赖无法编译的小代码片段，聚焦于二分类（是否存在漏洞），且未考虑公开发布的数据集可能已成为模型训练语料的风险。我们推出了RustMizan，一个针对Rust漏洞分析的基准测试框架，以弥补这些不足。RustMizan包含在 crate、文件和函数级别可编译的代码变体，并带有二分类漏洞检测、CWE分类以及函数和行级别定位的标注。配套的突变框架能生成语义保持的代码变体，用于污染测试和鲁棒性探测。在具有命令行访问权限的智能体设置中，对四个前沿模型的测试显示，二分类准确率在 56-65% 之间，但行定位的 F1 值仍接近 20%，且对抗性提示会使行级 F1 下降约 27%。

## Abstract
LLM agents are increasingly applied to vulnerability analysis, but existing benchmarks have not kept pace. They typically rely on small non-compilable snippets, focus on binary classification (vulnerable or not), and do not account for the risk that publicly-released datasets are part of model training corpora. We introduce RustMizan, a benchmarking framework for Rust vulnerability analysis that addresses these gaps. RustMizan contains compilable code variants at the crate, file, and function levels, with annotations for binary vulnerability detection, CWE classification, and function- and line-level localization. A paired mutation framework produces semantics-preserving code mutants for contamination testing and robustness probing. Across four frontier models in an agentic setup with command-line access, binary classification sits in the 56-65% range, but line localization F1 stays near 20%, and adversarial cues drop line F1 by about 27%.

### 2. 论文提出的方法论
RustMizan框架的核心思想是：通过提供**高质量、可编译、多层级**的代码样本和**语义保持的代码突变**，构建一个能全面、鲁棒地评测漏洞分析能力的平台。
*   **数据集构建**:
    *   **多任务标注**: 为每个漏洞提供四类标注：二进制漏洞检测、CWE分类、函数级定位、行级定位。
    *   **多级可编译变体**: 从真实CVE中手动构建不同上下文规模的独立、可编译Rust项目，包括：
        *   **Crate级**: 完整的Rust包。
        *   **文件级**: 仅包含漏洞文件及其编译依赖。
        *   **函数级**: 仅包含漏洞函数及其编译依赖。
    *   这一设计使得LLM Agent可以使用shell、编译器（如`cargo`）和文件浏览器进行交互式探索，而非被动接受代码片段。
*   **污染缓解与鲁棒性测试框架**:
    *   **良性突变**: 应用不改变代码语义的重构，如循环重写、变量/函数重命名、格式化等，用于测试模型是否依赖记忆而非推理。
    *   **恶性突变**: 注入对抗性安全提示，如误导性注释（“// 此行已修复，安全”）或暗示安全的标识符（`safe_fn`），用于评估模型的鲁棒性。
    *   **Rust特有突变**: 使用Rust语言特有的功能进行语义保持的变更，如插入无法到达的`unsafe`块。
    *   **真值维护机制**: 为应对突变导致的代码行号、标识符改变，框架采用三种跟踪机制：标记跟踪（插入特殊注释标记）、内容跟踪（匹配漏洞行文本）、重命名验证（维护重命名映射表），确保突变后标注依然准确。

1.  **检测与定位能力脱节(RQ1)**: 前沿模型能较好地进行漏洞检测（准确率56-65%），但在精确定位（尤其是行级定位）上表现很差（F1值仅约20%），揭示出现有模型“知其然，不知其所以然”的现状。

*   **创新的基准框架**: 首次实现了可编译、多任务、多层级、污染感知的综合性漏洞分析基准，填补了领域空白。
*   **高质量数据集**: 手动构建和同行评议确保了极高的数据质量和标签准确性，克服了自动标注数据集噪声大的通病。

### 8. 不足与局限
*   **数据集规模有限**: 仅包含42个CVE，泛化能力有待更大规模数据集的验证。尽管这是质量优先的选择，但样本多样性受限。
*   **标签假设局限**: 将“修复后”的代码视为“无其他漏洞”是一种简化假设，可能与实际情况不符。
*   **突变覆盖率不均**: 不同代码变体因其语法结构不同，能应用的突变操作数量不一，可能导致污染缓解效果在不同样本上不均衡。
*   **污染的持续性挑战**: 论文承认，一旦突变后的数据集公开，未来仍可能被纳入训练语料，这是一个固有的、需持续对抗的挑战。
