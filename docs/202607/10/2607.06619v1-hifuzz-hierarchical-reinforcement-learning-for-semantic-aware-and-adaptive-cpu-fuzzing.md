## 摘要
由于传统基于变异的模糊测试效率低下，现代处理器验证难以达到深层的架构状态。我们提出HiFuzz，一种新颖的分层强化学习框架，用结构化的两层生成过程取代变异：程序智能体负责全局布局，基本块智能体负责精确的指令填充。为了克服奖励稀疏性，HiFuzz集成了自适应覆盖率奖励机制和提供内在反馈的语义感知基本块编码器。在三个真实RISC-V核心上的广泛评估表明，HiFuzz在覆盖率和漏洞检测方面显著优于最先进的模糊测试工具。

## Abstract
Modern processor verification struggles to reach deep architectural states due to the inefficiencies of traditional mutation-based fuzzing. We propose HiFuzz, a novel hierarchical reinforcement learning framework that replaces mutation with a structured, two-layer generation process: a Program Agent for global layout and a Basic Block Agent for precise instruction filling. To overcome reward sparsity, HiFuzz integrates an adaptive coverage reward mechanism and a semantic-aware basic block encoder providing intrinsic feedback. Extensive evaluations on three real-world RISC-V cores demonstrate that HiFuzz significantly outperforms state-of-the-art fuzzers in coverage and bug detection.

### 1. 论文的核心问题与整体含义
*   **核心问题**：现代处理器设计复杂度激增，传统基于变异的硬件模糊测试（如RFUZZ、DifuzzRTL）因盲目破坏指令语义和依赖随机启发式，难以有效探索深层微架构状态，存在**奖励稀疏、反馈延迟、覆盖偏差（掩蔽效应）** 三大挑战。
*   **整体含义**：提出 **HiFuzz**，通过分层强化学习将测试程序生成过程分解为全局结构规划与局部指令填充，并结合语义感知内在奖励与自适应覆盖奖励，实现更高效、更深的处理器功能验证，大幅超越现有最先进模糊器。

### 2. 论文提出的方法论
*   **核心思想**：采用**两层分层强化学习（HRL）** 替代传统变异，分离宏观程序结构与微观指令选择，并引入两种反馈机制解决奖励稀疏与覆盖偏差。
*   **关键技术细节**：
    *   **可配置测试生成器**：将程序生成分为全局配置（内存布局、基本块数量、控制流）和基本块级配置（指令类别分布、终止模式），确保生成程序语法和语义正确。
    *   **双智能体架构**：
        *   **程序智能体（Program Agent）**：基于 **Rainbow DQN**，根据生成统计的11维状态，选择1,375种离散全局配置。
        *   **基本块智能体（Basic Block Agent）**：基于 **PPO** 多头部Actor-Critic，输出指令类别分布与终止信号，将具体指令合成委托给约束生成器，由内在和外在奖励两支价值头部分别学习。
        *   内在奖励 \( R_{int} \) 定义为生成基本块嵌入与在线聚类中心的最小余弦距离，提供密集学习信号。
        *   外在奖励 \( R_{ext} \) 基于归一化覆盖率提升。
        *   双优势Actor损失：\( L_{actor} = \alpha_{int} \hat{A}_{int} + \alpha_{ext} \hat{A}_{ext} \)。
    *   **语义感知基本块编码器（Semantic-Aware BB Encoder）**：
        *   采用结构化分词器将指令分解为操作数角色、执行单元元数据等语义标记。
        *   编码器骨干为**Bi‑LSTM**（96维输出），训练包含两阶段：① 自监督掩码语言模型（令牌级和指令级掩码）学习语法；② 监督相似度对齐学习，通过自定义的硬件感知基本块相似度指标 `BB-Sim`（融合加权序列比对、指令类型重叠、依赖结构）进行对比学习，使得嵌入余弦相似度逼近该指标（最终皮尔逊相关系数达0.876）。
    *   **自适应覆盖率奖励机制（ACRM）**：
        *   将DUT划分为M个独立模块，得到模块级覆盖率。
        *   将模块选择建模为多臂赌博机，用**UCB**公式动态计算权重 \( w_i^{(t)} = \text{softmax}\left( \mu_i^{(t)} + \beta(t) \cdot (1 - \frac{\text{cov}_i^{(t)}}{\max\text{cov}_i}) \right) \)，平衡利用与探索。
        *   最终外在奖励为模块覆盖率增益的加权和，迫使智能体关注欠验证模块，消除掩蔽效应。

*   **数据集/场景**：三个真实RISC‑V处理器核心——**Rocket**（顺序）、**BOOM**（乱序）和**CVA6**（SystemVerilog实现，用于跨生态验证）。
*   **Benchmark 与对比方法**：
    *   基线：DiffuzzRTL（变异）、ProcessorFuzz（CSR引导的变异）、Cascade（随机构造生成）。

*   **消融实验**：禁用ACRM、禁用BB编码器后与“DQN+PPO”双智能体基线对比，量化各组件贡献。

### 6. 论文的主要结论与发现
*   HiFuzz在24小时内，于Rocket核心上的**控制寄存器覆盖率达1,102,343**，较Cascade提升**48.9%**，单测试覆盖效率提升**3.3倍**。
*   在CVA6（非Chisel核心）上总覆盖率（线+翻转）比Cascade高**7.2%**，证明方法具有跨实现通用性。
*   模块级分析显示ACRM能有效将探索重心从易饱和模块转向硬覆盖模块（如FPU）。
*   漏洞检测方面，在Encarsia基准上，HiFuzz于BOOM和Rocket共检出**30/60个漏洞**，超过Cascade的26个，尤其在复杂乱序核心上优势更明显。
*   生成程序质量指标（有效指令占比约96.7%，平均依赖链长度18.5）远超变异类和随机构造类模糊器。

### 7. 优点
*   **方法创新**：将测试程序生成形式化为分层MDP，巧妙解决了动作空间爆炸与结构依赖问题；语义感知编码器为无仿真反馈的指令级决策提供了高质量内在奖励；UCB动态加权有效解决了覆盖的掩蔽效应。
*   **设计务实**：Bi‑LSTM编码器选择经过充分对比，更适配基本块短序列特性，且模型小巧（0.69M参数）；智能体输出高层配置而非具体指令，确保文法正确性，化解“控制-合法性”两难。
*   **评估严谨**：覆盖三个异构DUT，并使用与当前最优的生成式模糊器Cascade相同的实验平台，增强了结果可信度。

### 8. 不足与局限
*   **应用范围限制**：仅针对单核（无A扩展）程序生成，无法处理多核一致性、内存序等并发漏洞；且当前实现深度绑定RISC‑V，移植到x86/ARM等复杂ISA需重写大量语义规则。
*   **依赖模块级信号**：ACRM需要模块级覆盖率信号，若只有全局覆盖率，机制会退化为全局奖励，削弱其优势。
*   **实验重复性**：因24小时完整模糊测试代价高昂，未进行多随机种子重复实验，无法量化不同随机性导致的运行间方差。
*   **算力开销**：相较于非学习类生成器（如Cascade），HiFuzz引入了RL训练和编码器推理开销（生成时间占比30.5%），虽然提升了总效率，但在极端时间预算下可能优势减弱。
