## 摘要
图数据库管理系统（GDBMS）是管理互联数据的关键基础设施。现有的GDBMS测试方法主要依赖差分测试和蜕变测试。这些方法的结果一致性预言将输入限制为可在不同引擎或转换间比较的查询，导致单个引擎的运行时故障（如崩溃和内存错误）未被充分探索。为GDBMS开发专用模糊测试工具面临两个关键挑战：（1）在复杂的图约束下生成有效且结构多样的查询；（2）引导探索以捕捉拓扑相关的执行行为。为解决这些挑战，我们提出GRAF，一个面向GDBMS查询引擎的黑盒模糊测试框架。首先，GRAF引入了基于级联依赖解析的图上下文感知查询生成。它通过根据当前图状态联合解析标签、关系类型、属性、值和变量作用域，实例化由大语言模型（LLM）生成的参数化Cypher骨架。该过程生成结构多样的查询，同时消除语法和语义违规。其次，GRAF应用五种图特定的变异算子，这些算子由执行状态反馈（包括执行时间、结果大小和系统状态）引导。这些反馈使探索远离无产出的查询和代价高昂的遍历，同时优先对异常执行附近的局部变异。我们在六种广泛使用的GDBMS上将GRAF与三种现有方法进行了评估。在每个目标上，GRAF始终比最强基线提高31.6%至41.1%的行覆盖率。在12小时的模糊测试中，它触发了25个独特的bug，而所有基线总计只有6个。总体而言，GRAF发现了34个此前未知的bug，其中32个已被开发者确认，23个分配了CVE编号。

## Abstract
Graph Database Management Systems (GDBMSs) are essential infrastructure for managing interconnected data. Existing GDBMS testing methods primarily rely on differential and metamorphic testing. The result consistency oracles of these methods constrain inputs to queries that are comparable across engines or transformations, leaving single engine runtime failures, such as crashes and memory errors, insufficiently explored. Developing dedicated fuzzers for GDBMSs faces two key challenges: (1) generating valid and structurally diverse queries under complex graph constraints, and (2) guiding exploration to capture topology dependent execution behavior. To address these challenges, we propose GRAF, a black box fuzzing framework for GDBMS query engines. First, GRAF introduces graph context aware query generation based on cascading dependency resolution. It instantiates parameterized Cypher skeletons generated by a Large Language Model (LLM) by jointly resolving labels, relationship types, properties, values, and variable scopes against the active graph state. This process produces structurally diverse queries while eliminating syntactic and semantic violations. Second, GRAF applies five graph specific mutation operators guided by execution state feedback, including execution time, result size, and system status. This feedback steers exploration away from unproductive queries and expensive traversals, while prioritizing local mutations around abnormal executions. We evaluated GRAF against three existing approaches on six widely used GDBMSs. GRAF consistently improves line coverage by 31.6% to 41.1% over the strongest baseline on each target. In 12 hour fuzzing, it triggered 25 unique bugs, compared to six from all baselines combined. Overall, GRAF discovered 34 previously unknown bugs, with 32 confirmed by developers and 23 assigned CVEs.

## 论文核心问题与整体含义

*   **研究动机**：图数据库管理系统（GDBMSs）作为管理高连接数据的核心基础设施，广泛应用于金融欺诈检测、社交网络分析等场景。其可靠性至关重要。然而，现有GDBMS测试方法（如差分测试、蜕变测试）主要依赖结果一致性预言，迫使生成的查询必须跨引擎或通过语义等价变换可比较。这使探索的输入空间偏向于“产生结果”的查询，而单引擎运行时故障（如崩溃、内存损坏、资源耗尽）未被充分测试。
*   **核心问题**：为GDBMS开发专用模糊测试器面临两大挑战：（1）**查询生成**——在标签、关系类型、属性、变量作用域等复杂的图约束下，生成语法正确且结构多样的有效查询；（2）**执行引导**——图数据的高不规则拓扑（中心节点、稠密子图、环路）导致执行行为对拓扑极度敏感，传统的代码覆盖率反馈无法捕捉这种数据流驱动的执行代价（例如，遍历深度微调可能导致执行时间指数级爆炸，但控制流路径相似）。因此，需要新的反馈机制来引导探索，避开无果或代价过高的查询。

## 论文提出的方法论

论文提出 **GRAF**，一个黑盒、反馈驱动的图数据库模糊测试框架，核心思想是：**将抽象查询结构与具体图数据解耦，并借助执行状态反馈引导变异**。框架由两个主要组件构成：

**1. 图上下文感知的查询生成（Graph Context Aware Query Generation）**  
- **图上下文建模**：初始化目标数据库后，抽取形式化上下文\( C = (\mathcal{M}, \mathcal{G}, \mathcal{V}) \)。其中，\(\mathcal{M}\) 记录模式元数据（标签、合法关系三元组、属性及其数据类型）；\(\mathcal{G}\) 描述全局拓扑（节点数、边数、密度、直径估算）；\(\mathcal{V}\) 提供类型特定的值（包括采样自真实数据的观测值、边界值和异常值）。  
- **LLM驱动的骨架生成**：利用大语言模型（如 Gemini‑3.1‑pro）一次性合成约5000个参数化Cypher骨架。骨架中强制使用标准化变量名（如 `n1`, `r1`）和带索引的占位符（如 `$LABEL_i`），并明确表达变量作用域。生成后通过占位符校验、ANTLR语法解析、作用域分析和结构去重进行过滤。  
- **级联依赖解析实例化**：将骨架绑定到图上下文时，按照依赖链顺序分三阶段解析：  
  1. **拓扑解析**：从模式元数据\(\mathcal{M}\)中抽样源节点标签，据此选出合法关系类型，再确定目标节点标签，从而构建一个在图中真实存在的图模式。  
  2. **属性与值解析**：对每个实体，从其合法属性集中选取属性名，再利用值构造器\(\mathcal{V}\)生成与属性类型匹配的取值（可采样真实数据或注入边界值）。  
  3. **全局参数对齐**：根据拓扑规模\(\mathcal{G}\)设定 `LIMIT`、遍历深度等参数。  
  该过程确保生成的查询在拓扑和语义上合法，同时保留骨架的结构多样性。

**2. 执行状态指导的查询变异（Execution State Guided Query Mutation）**  
- **五种图特定变异算子**：在保持上下文有效性的前提下，对查询语法树应用五种变异：  
  *拓扑变异*（插入/移除/重连 `MATCH` 中的模式）、  
  *深度与体积变异*（调整遍历边界、`LIMIT` 等控制中间结果大小的参数）、  
  *谓词变异*（插入/替换/删除/松弛 `WHERE` 条件）、  
  *子句插入*（注入 `WITH`、聚合、子查询等流水线成分）、  
  *值变异*（将字面量替换为观测值、边界值或异常值）。  
- **反馈引导机制**：每个查询执行后，监视器记录动态反馈向量\( F = \langle t, |R|, s \rangle \)（执行时间、结果大小、终止状态）。基于此引导后续变异：  
  * 快速且结果为空 → 提高谓词松弛、拓扑调整和插入观测值的概率；  
  * 高代价或超时 → 惩罚增加深度或体积的变异，倾向有界缩减或其他组件的变异；  
  * 异常终止（崩溃、断言失败、sanitizer 报警等）→ 保留该查询并优先进行局部变异以定位故障子句。  
  整个过程完全不依赖内部代码覆盖率，完全利用外部可观察状态持续将探索导向深层引擎逻辑。

## 论文的主要结论与发现

1. **覆盖率显著提升**：GRAF在所有六个目标上均取得最高的行覆盖率，相比每目标的最强基线提升31.6%–41.1%（如对比Dinkel覆盖多出31.6%–77.3%），且贡献了大量的独特覆盖行，表明触达了更深的引擎内部逻辑。
2. **有效性与多样性平衡**：GRAF的查询有效率在61.8%–76.7%之间，虽略低于保守的Dinkel，但远高于BUZZBEE和AFL++，能够在保证语法语义正确的前提下保持结构高度复杂和多样性。
3. **组件不可或缺**：消融实验显示，去除骨架实例化器使VQR从71.3%骤降至10.7%，行覆盖下降81%–86%；去除执行监视器虽VQR仅降5%，但行覆盖下降24%–31%，且超时/慢查询数量增加约一倍，证明反馈机制有效抑制了查询爆炸。
4. **Bug发现能力卓越**：在12小时受控实验中，GRAF独自触发25个独特Bug，而Dinkel、BUZZBEE和AFL++分别触发4、2、0个，GRAF发现的Bug超过所有基线之和。全周期内共发现34个此前未知的Bug，32个被开发者确认，23个获得了CVE编号，涵盖了模式匹配、路径扩展、聚合、属性访问、事务管理等核心组件。

## 优点

*   **创新的解耦设计**：将查询的抽象结构（由LLM生成多样骨架）与具体图数据绑定（通过级联依赖解析）完美分离，既保证了查询的结构多样性，又确保了语义与拓扑正确性，有效解决了传统生成器多样性不足和变异器无效率高的问题。
*   **实用且轻量的反馈机制**：摒弃了传统代码覆盖率，完全采用执行时间、结果集大小和系统状态等黑盒信息引导变异，能准确规避无果查询和指数级遍历爆炸，将算力聚焦于挖掘深层引擎逻辑，这一设计特别适合拓扑敏感的图数据库。
*   **全面的评估与显著的实战效果**：在六款主流系统上的对比与消融全面、严谨，覆盖率和Bug发现的双重优势令人信服。23个CVE的产出直接证明了工具的高度实战价值和威胁检测能力。
*   **低成本高扩展性**：LLM仅用于一次性骨架合成，成本可忽略不计，且框架通过适配器支持多种图数据库，具有较强的泛化潜力。

## 不足与局限

*   **查询语言与数据库范式的限制**：当前GRAF仅针对Cypher查询语言，虽然其设计思想可移植，但未在基于Gremlin或其他特种图语言的系统上验证。论文仅关注运行时稳定性Bug（崩溃、内存错误），尚不能检测逻辑错误（如错误的结果集），其错误发现范围仍有边界。
*   **初始图状态的依赖**：方法依赖框架自建的初始图及其导出的上下文。该图虽然设计了多种拓扑模式，但其是否能充分反映真实应用数据的复杂性和极端分布仍有待验证；若图状态过于简单或与目标引擎优化策略不匹配，可能限制探索深度。
*   **骨架生成对LLM的隐性偏置**：骨架多样性受限于所用LLM的训练数据与提示设计，可能偏向常见查询模式，难以穷举某些冷门或特定方言的语法结构，从而导致某些代码区域始终无法触达。
*   **反馈粒度的粗糙性**：执行时间、结果大小和状态属于宏观反馈，虽然能有效避开路径爆炸，但可能对某些需要更精细信号（如内部算子的数据流分布、内存分配模式）的复杂逻辑探索不足，仍可能错失部分需要精准条件触发的缺陷。
