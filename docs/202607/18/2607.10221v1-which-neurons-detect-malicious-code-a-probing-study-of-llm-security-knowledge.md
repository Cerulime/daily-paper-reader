## 摘要
背景：大型语言模型（LLMs）在理解和生成源代码方面日益强大，因此被广泛用于代码补全、修复和漏洞检测等软件工程任务。然而，尽管它们在经验上表现良好，LLMs识别恶意或易受攻击代码模式的内部机制仍然知之甚少。目的：我们研究了恶意软件检测行为在LLMs前馈网络（FFN）神经元中的编码位置，并通过对已识别神经元进行因果干预来验证归因，旨在找出检测恶意代码最重要的神经元。方法：我们应用机制可解释性方法，在三个经过指令微调的LLMs：Llama3.1-8B-Instruct、Mistral-v0.3-7B-Instruct和Qwen2.5-7B-Instruct中定位负责恶意软件检测行为的神经元。使用来自PyPI Malregistry的1500个恶意和1500个良性PyPI包，我们将行为归因于一组神经元。结果：实验结果表明，放大促进恶意软件检测的神经元同时抑制抑制性神经元可以提高准确率，而反向操作则会使预测坍缩到单一类别，尽管效应的幅度和一致性在很大程度上依赖于模型。我们证明了不同模型的安全防护检测机制存在差异，每个模型在其FFN层中以不同方式表示其恶意软件检测行为。结论：探测与安全相关知识相关的神经元有助于我们深入了解LLMs如何编码恶意编程概念，识别潜在的有害记忆行为，从而为更可靠的防御机制铺平道路，例如神经元级编辑、选择性遗忘以及针对代码LLMs的安全感知对齐。

## Abstract
Background. Large language models (LLMs) have become increasingly capable of understanding and generating source code, leading to their widespread adoption in software engineering tasks such as code completion, repair, and vulnerability detection. However, despite their strong empirical performance, the internal mechanisms through which LLMs recognize malicious or vulnerable code patterns remain poorly understood. Aim. We investigated where the malware detection behavior is encoded inside LLMs Feed Forward Network (FFN) neurons and verified the attribution with causal interventions on the neurons identified. This aims to identify the most important neurons in detecting malicious code. Methods. We applied mechanistic interpretability methods to locate the neurons being responsible for malware-detection behavior in three instruction-tuned LLMs: Llama3.1-8B-Instruct, Mistral-v0.3-7B-Instruct, and Qwen2.5-7B-Instruct. Using 1,500 malicious and 1,500 benign PyPI packages from the PyPI Malregistry, we attribute the behavior to a set of neurons. Results. The experimental results reveal that amplifying facilitating neurons for malware detection while suppressing inhibiting ones can boost accuracy, while the reverse collapses predictions toward a single class, although the magnitude and consistency is heavily model-dependent. We demonstrated that the guardrail detection mechanism varies across models, each represents its malware detection behavior differently within its FFN layers. Conclusions. Probing the neurons associated with security-relevant knowledge helps us gain insights into how LLMs encode malicious programming concepts, identify potentially harmful memorized behaviors, paving the way toward more reliable defense mechanisms, such as neuron-level editing, selective unlearning, and security-aware alignment for code-focused LLMs.

### 1. 论文的核心问题与研究动机  
- 大型语言模型（LLMs）在代码生成、漏洞检测等软件工程任务中表现出色，但其识别恶意代码的内部工作机制仍是“黑箱”。  
- 当前安全防御多依赖提示工程或后过滤，难以应对对抗性攻击，根源在于缺乏对LLMs内部安全知识表示的理解。  
- 研究目标：**定位LLM前馈网络（FFN）中负责检测恶意代码的神经元**，并利用因果干预验证其功能，为透明化、神经元级编辑和选择性遗忘等可靠防御奠定基础。

### 2. 方法论  
整体流程分为四个阶段：  
- **数据集构建与代理提示**：  
  - 从 PyPI Malregistry 收集 1500 个恶意包和 1500 个结构相似的良性包，聚焦安装时/导入时攻击载荷。  
  - 将代码片段转化为二元多选题（问是否是恶意代码），并通过**标签交换**设计“代理问题”以消除模型对选项位置的偏好。  
- **神经元归因**：  
  - 目标函数 \(F(x) = Z_{\text{correct}} - Z_{\text{incorrect}}\)，使归因同时奖励正确选项并惩罚错误选项。  
  - 采用两种机制可解释性方法：  
    1. **集成梯度 (IG)**：沿基线到输入的插值路径累积梯度，聚焦于每一层 FFN 神经元的激活值。  
    2. **神经元电导 (NC)**：扩展 IG，衡量隐藏神经元从输入到输出的“信号传导”大小。  
- **评分与神经元筛选**：  
  - 为每个样本计算“示例得分”（ES），选取得分最高和最低的神经元分别作为“促进者”（G）和“抑制者”（B）。  
  - 过滤在样本间角色矛盾的模糊神经元，计算聚合电导得分（ACE），最终选取 ACE 最高和最低的 100 个神经元作为“好神经元”和“坏神经元”。  
- **因果干预**：  
  - 在选定的神经元上施加五种乘法干预条件（缩放因子）：  
    - 基线（不修改）、抑制好神经元（sG=0）、抑制坏神经元（sB=0）、增强器（sG=2, sB=0）、降级器（sG=0, sB=2）。  
  - 观察准确率、F1 等指标的变化，通过效果差距 \(\Delta = \text{Accuracy}_{\text{Enhancer}} - \text{Accuracy}_{\text{Degrader}}\) 衡量神经元因果重要性。

### 3. 实验设计  
- **数据集**：  
  - 来源：PyPI Malregistry 的安装时/导入时攻击负载，1500 恶意 + 1500 良性，按行数桶分层分割。  
  - 测试集：2890 样本（1445 恶意 + 1445 良性），归因用子集 110 样本。  
- **模型**：  
  - Llama3.1‑8B‑Instruct (8B, 32 层)、Mistral‑v0.3‑7B‑Instruct (7B, 32 层)、Qwen2.5‑7B‑Instruct (7B, 28 层)。  
- **对比方法**：  
  - 归因方法：集成梯度 vs. 神经元电导。  
  - 干预条件：五种干预倍率（基线、Supp‑G、Supp‑B、Enhancer、Degrader）。  
- **评估指标**：准确率（含95%置信区间）、每类 F1、McNemar 检验评估干预间差异显著性、效果差距 Δ。

### 5. 实验数量与充分性  
- **主要实验**：3 个模型 × 2 种归因方法 × 5 种干预条件，评估准确率、F1 等，并计算效果差距与 McNemar 检验。

### 6. 主要结论与发现  
- 放大促进恶意检测的神经元并抑制抑制性神经元**可以提升准确率**，反之（抑制好神经元或放大坏神经元）导致预测坍缩至单一类别；效应的幅度和一致性**高度依赖模型**。  
- 不同模型将恶意检测信号集中在**不同的层带**：Llama3.1 和 Mistral 在中间层，Qwen2.5 在后期层；且“好”与“坏”神经元倾向于共同定位。  
- 被选出的**个体神经元不稳定**（Jaccard < 0.2），但**层级分布较稳定**（层数余弦相似度 0.6–0.9）；层级稳定性与因果干预效果存在共现关系。  
- 安全知识编码在特定神经元群中，为轻量级提前退出安全检查、靶向模型编辑、层特定遗忘等防御策略提供了实证基础。

### 7. 优点与亮点  
- 首次利用机制可解释性定位代码 LLM 中恶意检测的神经元，并**通过因果干预严格验证**，而非仅停留于关联分析。  
- 实验设计细致：代理提示消除位置偏见，双目标函数、模糊神经元过滤、多种干预范围，使得归因较为可靠。  
- 发现了**层级偏好分布**和**层级稳定性强于个体稳定性**的现象，对后续编辑/加工有指导意义。  
- 提供完整**复制包**，促进开放科学。

### 8. 不足与局限  
- **模型规模受限**：仅测试 7‑8B 参数级模型，未覆盖更大规模代码 LLM，结论可能无法泛化。
