## 摘要
LLM辅助的逆向工程(RE)系统分析来自攻击者控制的二进制文件衍生出的字符串、反编译器输出和工具报告。二进制文件可以将数据伪装成指令，或者将同一来源的记录伪装成独立证据。我们将此类失败称为逆向工程中的表示混淆攻击(RARE)：流水线将正确提取的观察错误地提升为指令权威、声明验证证据或可信分析状态，却没有相应的权限或支持。RARE-Bench用行为检查的干净二进制和对抗性二进制来衡量这些失败。经过一项探索性的11,520次调用研究后，我们在20个新程序和两个模型上测试了RARE-Guard的授权和证据控制。在没有运行时控制的情况下，模型在35/40的对抗性案例中提出植入的不安全操作，在0/40的干净案例中则没有。当仅以数据形式展示二进制衍生内容（仅数据渲染）时，它们仍会提出15个不安全提议。工具授权拒绝了所有15个提议，并授权了所有40个匹配的分析师请求。在面对相同的报告草稿时，支持门通过单独计算一个来源的记录，验证了23/40的错误声明。来源门在计算支持之前对这些记录进行分组，验证了0/40的错误声明，并保留了全部40个有支持的声明。我们随后在另外16个程序上对Ghidra、r2pipe和angr进行了插桩。在预选的8个程序子集中，没有单一工具草稿达到支持门对错误声明的验证阈值。在所有16个程序的融合草稿中，支持门验证了32/32个错误声明。来源门阻止了所有32个声明的验证，并保留了全部32个有支持的声明。确定性渲染器防止降级的声明重新出现在最终报告中。因此，二进制衍生内容可以在不获得高于工具的权限的情况下指导分析，而来自多个工具的视图并不一定提供独立证据。

## Abstract
LLM-assisted reverse-engineering (RE) systems analyze strings, decompiler output, and tool reports derived from ttacker-controlled binaries. A binary can make data look like instructions or records from one origin look like independent evidence. We call such failures Representation-Confusion Attacks in Reverse Engineering (RARE): the pipeline promotes a correctly extracted observation to instruction authority, claim-validating evidence, or trusted analysis state without the authority or support that role requires. RARE-Bench measures these failures with behavior-checked clean and adversarial binaries. After an exploratory 11,520-call study, we test RARE-Guard's authorization and evidence controls on 20 new programs and two models. Without runtime controls, the models propose a planted unsafe action in 35/40 adversarial cases and 0/40 clean cases. When binary-derived content is shown only as data (Data-Only rendering), they still make 15 unsafe proposals. Tool Authorization denies all 15 and authorizes all 40 matched analyst requests. On identical report drafts, Support Gate validates 23/40 false claims by counting records from one origin separately. Provenance Gate groups those records before counting support, validates 0/40 false claims, and retains all 40 supported claims. We then instrument Ghidra, r2pipe, and angr on 16 further programs. In a preselected eight-program subset, no single-tool draft reaches Support Gate's validation threshold for the false claim. In fused drafts across all 16 programs, Support Gate validates 32/32 false claims. Provenance Gate prevents validation of all 32 and retains all 32 supported claims. A deterministic renderer prevents downgraded claims from reappearing in the final report. Binary-derived content may therefore guide analysis without gaining authority over tools, and views from several tools do not necessarily provide independent evidence.

### 1. 论文的核心问题与整体含义
- **核心问题**：LLM辅助的逆向工程（RE）系统需要分析来自二进制文件的各种数据（字符串、反编译器输出、工具报告等）。攻击者可以在受控的二进制文件中嵌入恶意内容，使这些数据被LLM赋予不正当的“角色”，从而误导分析流程。
- **攻击定义**：论文将此类攻击定义为**表示混淆攻击（RARE）**，即流水线将正确提取的观察“提升”为指令权威、声明验证证据或可信分析状态，而没有相应的权限或支持。
- **威胁实例**：近期安全报告中已出现类似攻击（如macOS.Gaslight植入物伪造系统消息、恶意软件嵌入提示注入、供应链攻击中利用非执行头污染AI分析），表明攻击者正试图利用这一安全边界。
- **研究目标**：构建基准测试（RARE-Bench）来 **测量** 观察被赋予不当角色的程度，并设计运行时防御机制（RARE-Guard）来 **阻止** 这种不当“提升”，确保二进制派生内容只能作为数据引导分析，而不能获得高于工具的权威。

### 2. 论文提出的方法论
RARE-Guard 采用分层运行时控制，在数据和指令之间建立明确的安全边界。

- **核心思想**：
  - 区分“提取正确性”与“角色授权”。数据被正确提取不等于它拥有指令权威或独立证据地位。
  - 使用可信的**侧车**记录（sidecar）存储每条记录的来源、依赖、污点等元数据，模型仅看到紧凑的类型化视图及句柄，无法修改元数据。
  - 在不同阶段施加策略：工具操作授权、声明验证、状态持久化。

- **关键技术细节（四层防御）**：
  - **P0: 仅数据渲染（Data-Only）**：将二进制派生内容标记为数据，与分析员指令分离，但不能直接执行操作。
  - **P1: 工具授权（Tool Authorization）**：模型提出类型化工具提议（工具ID、参数、目的、引用句柄）。运行时中介检查**允许列表、参数模式、授权许可和资源限制**。拒绝原始shell访问、攻击者指定的执行路径等。授权决策在模型外确定。
  - **P2: 支持门（Support Gate）**：对声明应用确定性规则。检查引用记录的支持类型（文本、结构、行为）和可达性是否满足声明要求。若不满足，将声明降级为假设或状态不足。
  - **P3: 来源门（Provenance Gate）**：在P2基础上，对引用记录进行分组。**精确根规则**将来自同一点二进制位置或运行时事件的记录视为一个来源单元；**共同祖先规则**进一步将具有不同本地根但追溯到同一公共祖先的记录合并。分组后重新应用P2的支持规则，防止将重复视图误认为独立证据。
  - **P4: 状态门（State Gate）**：在写入和检索状态（如笔记、假设）时，保留状态、污点和支持引用。禁止在没有新支持的情况下将污点内容重新提升为已验证状态。
- **最终渲染**：确定性渲染器仅从后门控声明对象生成最终报告，防止自由文本字段绕过门控决策。

实验分为三个阶段，设计层次递进，分别探索攻击面、隔离机制和实际工具工作流。

    - **受控留出法研究**：20个新程序。

- **攻击有效性**：在Flat渲染下，模型在87.5%的对抗性案例中诱导出不安全提议（35/40），干净案例为0%；Data-Only渲染可减少约50%的诱导，仍有15个不安全提议。

### 7. 优点
- **问题定义清晰**：将攻击类从“提示注入”拓展至更广义的“表示混淆”，覆盖权威、证据和状态三种不当提升，具有前沿性。
- **方法论系统性强**：RARE-Guard的四层防御体系（数据渲染、授权、支持/来源门、状态门）分层解耦，设计精巧。利用可信侧车和类型化句柄，在模型不可信前提下实施强制访问控制。
- **实验设计严谨**：采用合成但行为可验证的程序，建立了包含干净、控制、对抗性的配对基准，排除了混淆因素。通过多阶段实验（探索、隔离、验证）逐层深入，内部效度高。
- **可重复性与透明性高**：提供代码、二进制、记录和数据，冻结所有参数，支持离线验证。使用固定草案重放来隔离策略效应，消除模型随机性影响。

### 8. 不足与局限
- **实验室与实际环境差距**：
  - 程序均为合成的、非恶意的ELF/x86-64样本，不含实际恶意软件行为，难以证明其在真实多样恶意软件中的普适性。
  - 任务受限，如两轮工具执行、固定证据融合，非开放式的自主代理探索。
- **模型与工具的有限性**：探索性阶段后的实验仅使用两个固定版本的专有模型，结果可能不适用于其他或更新的模型。工具集限定于Ghidra、r2pipe、angr，未覆盖更多逆向工具。
- **来源依赖“已知”假设**：来源门仅能分组**已记录在侧车中**的依赖关系，无法发现未记录的依赖或证明剩余单元的独立性。
- **状态门效应未充分检验**：由于无模型草稿请求无支持的升级，状态门(P4)对阻止不当升级的因果效应未得到实证，仅验证了状态保持功能。
- **特定目标与度量**：评估基于注册的特定“目标命题”和固定模式，可能无法覆盖所有形式的隐蔽误导或长期记忆漂移。
